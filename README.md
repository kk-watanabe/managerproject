# README

## 開発管理統合アプリケーション（PoC）
企業実習で出された課題に取り組んで作成した社内開発管理アプリです。実習は作業時間が最大100時間という制限の中で、要件の読み込み、設計からテストまで一貫して行いました。<br>

<br>

開発管理に関するプロセスとデータを一元管理するアプリケーションです。従来の管理方式の課題として、申請システム、各部門での進捗管理（Excel）、本部への共有、予算管理がそれぞれ分断され非効率だったことがありました。そこで、それらを「一元管理」することができれば、管理の効率化と可視化を実現できると考えました。本アプリはその仮説に対するPoCです。<br>
一つのアプリ上で案件申請、承認フロー、案件の進捗管理、予算実績管理をすべて管理することができます。

## 主な機能一覧
【申請者の機能】
- 新規案件の申請
- 自身の申請案件の進捗入力・更新
- 予算実績の入力
- 自身の案件一覧・ステータス確認

【部門管理者の機能】
- 申請の一次承認
- 自部門の案件一覧・進捗確認

【本部管理者の機能】
- 関連部門全案件の一覧・進捗確認
- 関連部門予算消費状況の確認

【その他の機能】
- 通知機能（予算超過、承認フローの審査結果）
- コメント機能

## 工夫点
* ### ロール別画面構成
各ロールに応じて別々の画面や機能を構成することで操作迷子を減らし、セキュリティを高めました。

* ### 細部のバリデーション設定や入力ミス防止のUI設計
記入漏れや日付選択の矛盾などの人為的なミスに対して、バックエンドからのエラー表示を行うことでデータの完全性を確保するようにしました。

* ### 予算超過や遅延発生時のアラート機能
予算超過や案件進捗の遅延が発生した際には、自動的に管理者に通知が送られます。管理者の早期対応を支援します。

* ### 進捗率バー表示
進捗率をアナログ表示にすることで、案件一覧で大量のデータが並ぶ中でも案件進捗を一目で認識できます。

* ### 承認待ち案件数や未読通知数をバッジ表示
リンクに対して件数をバッジで添えることで視認性を向上し、新規申請や未読通知の確認漏れ防止と早期対応を支援します。

## 今後の展望
今後の展望として、従来の管理方式で分散していたデータの引継ぎを効率化することを考えています。今回のアプリの目的は分散していたシステムを一元化することですが、新規システムに移行するとき、過去のデータの引継ぎが重要な問題となると考えられます。企業実習中はその辺りの視点から設計することができなかったため、今後はCSVのimport/exportなど、データ移行効率化にとって便利な機能を実装したいと考えています。<br>
また、100時間という時間の制約があったためあまりUIに凝ることができなかったのですが、色分けによる視認性向上やホーム画面のダッシュボード化など、よりユーザー目線で見やすく安定感のあるUIの向上を図ろうと思います。
